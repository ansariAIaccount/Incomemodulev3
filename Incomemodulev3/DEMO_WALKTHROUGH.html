The 30-second architecture

Every layer is transparent. You can inspect the engine's math on every row, drill into every JE memo, and follow the trail from a builder edit to a DIU batch ID.

The core engine is pure. All state — instrument definition, schedule, JEs, breach log — flows through loan-module-engine.js as functions. The V3 HTML page is just an orchestrator around it, with Supabase for persistence and a small Node proxy for Investran.

Stage 1 → Stage 2 — Daily engine + journals

Click Send to Cashflow Generation, then Run Accounting. The engine builds a 1,827-row daily schedule and 50 monthly JE batches for a 5-year loan.

assets/shot_03_stage2_run.png Stage 2 · Ready · 50 JE rows · toast: "Run persisted to Supabase"

What the engine computes daily

  • Interest accrual — balance × coupon × day-count factor. ACT/360, ACT/365, ACT/ACT, 30/360, 30E/360.
  • RFR compounding — lookback, observation shift, lockout, daily compounding — when a fixings series is loaded (see §8).
  • PIK — capitalisation on a configurable schedule, tracked separately from cash interest.
  • EIR fee accretion — deferred fee pool amortised to income over life. Crystallises on derecognition. Framework-tagged (ASC 310-20 / IFRS 9 §B5.4).
  • ECL provisioning — Stage 1 (12-month) / Stage 2 (lifetime) / Stage 3 (net carrying). Automatic on covenant breach (see §6).
  • FX revaluation — daily / monthly / quarterly cadence for cross-border deals.
  • Hedge accounting — CFH effective vs ineffective split, reclassification on settlement, de-designation.
  • Modifications — substantial (10% PV test) → derecognise + new instrument. Non-substantial → carrying-value adjustment + P&L.

Breach lifecycle persistence

Every breach + cure cycle is a row in covenant_breach_log. When you set lastReportedValue back to compliant and re-run, the row toggles to cured. Full audit trail forever.

assets/shot_07_breach_history.png Stage 0 · Cascade · Covenant card · "Breach history (4)" accordion expanded with cycle log

Round-trip demo (in DevTools console)

// Trigger a breach
B.covenants[0].lastReportedValue = 4.10;
window.registerBuilderInstrument();
runAccounting();
// Toast: "Breach log · 1 active · 0 cured"

// Cure it
B.covenants[0].lastReportedValue = 2.80;
window.registerBuilderInstrument();
runAccounting();
// Toast: "Breach log · 0 active · 1 cured"
// The row is now permanently marked cured with a cure_date timestamp.

RFR fixings ingestion

On any Floating-rate interest component, the RFR Fixings Series widget lets you drag-drop a CSV of daily observed benchmark rates. The engine's compounded-RFR math (lookback / observation shift / lockout / daily compounding / weighted) consumes them.

assets/shot_08_rfr_fixings.png Stage 0 · IC card · RFR Fixings Series expanded with sparkline + row list

CSV format

date,rate
2026-01-15,0.0425
2026-01-16,0.0428
2026-01-17,0.0432
2026-01-20,0.0435

Auto-detects percent vs decimal (values > 1 are treated as %). Supports YYYY-MM-DD, DD/MM/YYYY, YYYY/MM/DD formats. Tab-separated works too. Optional third column: source (e.g. "Bloomberg BGN", "BoE", "ICE").

Merge semantics: Uploads append + merge on date. Uploading the same file twice doesn't duplicate — the second load updates the row. Idempotent all the way to the DB (UNIQUE on interest_component_id + fixing_date).

Stage 3 — Post to Investran DIU

The end of the pipeline: click Post JE to Investran. When the DIU proxy is wired up, this is a real live batch post. Otherwise it runs a 6-step simulator with a "DEMO MODE" banner.

Deploy runbook: See DIU_PROXY_QUICKSTART.md for 3 deploy paths — local dev (5 min), Render.com (10 min), or internal VM behind a firewall.
